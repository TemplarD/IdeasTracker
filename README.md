# IdeaTracker - Платформа для краудсорсинга идей

[![CI/CD](https://github.com/yourusername/ideatracker/actions/workflows/ci.yml/badge.svg)](https://github.com/yourusername/ideatracker/actions/workflows/ci.yml)

> Платформа для коллективной генерации, оценки и развития идей проектов

## 🚀 Быстрый старт

```bash
# Клонирование
git clone <repository-url>
cd IdeasTraker

# Копирование .env
cp .env.example .env

# Запуск через Docker
docker-compose up -d

# Или локально
./setup.sh  # Linux/Mac
# или
setup.bat   # Windows
```

**После запуска:**
- Frontend: http://localhost:3001
- Backend API: http://localhost:3000
- Swagger: http://localhost:3000/api/docs

📖 **Подробная инструкция:** [RUN.md](./RUN.md)

## 📋 Техническое задание

### 1. Общие сведения

### 1.1. Назначение системы
Веб-приложение для коллективной генерации, оценки и развития идей проектов. Пользователи могут публиковать идеи, оценивать их по различным критериям, формировать команды для реализации и привлекать инвесторов.

### 1.2. Целевая аудитория
- **Авторы идей** — пользователи, публикующие идеи
- **Оценщики** — пользователи, оценивающие и комментирующие идеи
- **Участники команд** — пользователи, присоединяющиеся к реализации идей
- **Инвесторы** — пользователи, готовые финансировать проекты
- **Администраторы** — управление платформой и модерация

### 1.3. Требования к развёртыванию
- **Веб-приложение** (основной режим)
- **Возможность запуска как веб-портал** (мульти-тенантная архитектура)
- **Адаптивный дизайн** (desktop, tablet, mobile)

## 2. Функциональные требования

### 2.1. Модуль «Идеи»

| Функция | Описание |
|---------|----------|
| Создание идеи | Заголовок, описание (rich-text), вложения (файлы, ссылки на презентации) |
| Редактирование | Автор может редактировать свою идею |
| Удаление | Мягкое удаление (soft delete) с сохранением истории |
| Категоризация | Теги, категории (программа, сайт, сервис и т.д.) |
| Статусы идеи | `Черновик`, `Опубликована`, `В работе`, `Реализована`, `Закрыта` |

### 2.2. Модуль «Оценивание»

| Критерий | Шкала |
|----------|-------|
| Интерес | 1–5 звёзд |
| Польза | 1–5 звёзд |
| Прибыльность (теоретическая) | 1–5 звёзд |
| **Итоговый рейтинг** | Средневзвешенное значение |

**Дополнительно:**
- Комментарии к идеям (древовидные, с возможностью ответа)
- Голосование за комментарии (like/dislike)
- Уведомления автору о новых оценках и комментариях

### 2.3. Модуль «Команды»

| Функция | Описание |
|---------|----------|
| Создание группы | Для каждой идеи может быть несколько групп |
| Роли в команде | Настраиваемые роли (Разработчик, Дизайнер, Менеджер, Инвестор и др.) |
| Вступление в команду | Заявка → Подтверждение лидером группы |
| Описание группы | Цель, план, сроки |
| Прогресс группы | Публикация обновлений (посты в ленте группы) |
| Выход из группы | Добровольный выход или исключение лидером |

### 2.4. Модуль «Инвесторы»

| Функция | Описание |
|---------|----------|
| Профиль инвестора | Информация о бюджете, предпочтениях, истории инвестиций |
| Заявка на инвестирование | Инвестор может предложить сумму для идеи/группы |
| Условия инвестирования | Доля в проекте, процент автору идеи, сроки возврата |
| Статус сделки | `Предложено`, `Обсуждение`, `Согласовано`, `Завершено` |
| Уведомления | Стороны получают уведомления о статусе сделки |

### 2.5. Модуль «Пользователи»

| Функция | Описание |
|---------|----------|
| Регистрация/Авторизация | Email + пароль, OAuth (Google, GitHub) |
| Профиль пользователя | Аватар, био, навыки, репутация |
| Репутация | На основе активности, оценок, успешных проектов |
| Уведомления | Email, push (в приложении), настройки уведомлений |

### 2.6. Модуль «Администрирование»

| Функция | Описание |
|---------|----------|
| Модерация контента | Проверка идей, комментариев на соответствие правилам |
| Управление пользователями | Блокировка, разблокировка, выдача прав |
| Аналитика платформы | Статистика по идеям, пользователям, активностям |
| Настройки портала | Для мульти-тенантной версии — кастомизация под портал |

## 3. Нефункциональные требования

### 3.1. Технические требования

| Требование | Значение |
|------------|----------|
| **Фронтенд** | React + TypeScript + Bootstrap 5 + Material Design |
| **Бэкенд** | Node.js (NestJS) или Python (FastAPI) |
| **База данных** | PostgreSQL (основная), Redis (кэш, сессии) |
| **Аутентификация** | JWT + Refresh Token |
| **Файловое хранилище** | Локальное или S3-совместимое |
| **API** | RESTful + WebSocket (для уведомлений в реальном времени) |
| **Тестирование** | Unit-тесты (Jest/pytest), Integration-тесты, E2E (Playwright) |
| **CI/CD** | GitHub Actions / GitLab CI |
| **Контейнеризация** | Docker, Docker Compose |

### 3.2. Требования к безопасности
- Хеширование паролей (bcrypt/argon2)
- Защита от XSS, CSRF, SQL-инъекций
- Rate limiting для API
- Валидация всех входных данных
- HTTPS в продакшене

### 3.3. Требования к производительности
- Время ответа API < 200 мс (95-й перцентиль)
- Поддержка до 10 000 одновременных пользователей
- Кэширование часто запрашиваемых данных

## 5. Требования к тестированию

### 5.1. Стратегия тестирования

| Уровень | Инструменты | Покрытие |
|---------|-------------|----------|
| **Unit-тесты** | Jest (Node.js) / pytest (Python) | > 80% бизнес-логики |
| **Integration-тесты** | Supertest / httpx | Все API endpoints |
| **E2E-тесты** | Playwright | Критические пользовательские сценарии |
| **Нагрузочные тесты** | k6 / Apache JMeter | Пиковые нагрузки |

### 5.2. Критические сценарии для E2E

1. Регистрация → Создание идеи → Оценка → Комментарий
2. Создание команды → Вступление участника → Публикация прогресса
3. Заявка инвестора → Согласование → Заключение сделки

### 5.3. Непрерывное тестирование
- Запуск тестов при каждом PR
- Блокировка мержа при падении тестов
- Отчёт о покрытии в CI
